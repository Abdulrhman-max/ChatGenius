    Appointment Integration
    
            
    Sync appointments from your website or PMS with ChatGenius. When a patient books directly on your website, send the appointment to our API so the chatbot knows about it and avoids double-booking.
    

            
            
    
                How It Works
                
    

                    
    Overview
    
                    
    This integration allows your website or external PMS to push appointments directly into ChatGenius. Each company gets a permanent API key that never expires and never changes — use it in all your requests to authenticate which company the appointment belongs to.
    

                    
    Booking Flow
    
                    
    
                        
    1.Your website sends a POST request with patient info, doctor name, date, and time.
    
                        
    2.We validate the doctor exists in your account and the time is within the doctor's working hours for that day.
    
                        
    3.If the slot is free — the appointment is created and confirmed instantly. You get back a booking_id.
    
                        
    4.If the slot is taken — the patient is automatically added to the waitlist. An email is sent to the patient with two buttons:
    
                        
    
                            
    ●"Keep Me on Waitlist" — patient confirms they want to wait. If the main appointment is cancelled, they get moved in automatically.
    
                            
    ●"Remove Me from Waitlist" — patient is removed and won't be contacted again for this slot.
    
                        
    
                    
    

                    
    Time Handling
    
                    
    Send the start time only (e.g. 14:00 or 2:00 PM). The end time is calculated automatically based on the doctor's appointment length setting. Supports both 24-hour and 12-hour formats.
    

                    
    Working Hours Validation
    
                    
    Every request is validated against the doctor's schedule. If the doctor has a flexible schedule, we check the specific day's hours. If fixed, we use the general working hours. Requests on days off or outside working hours are rejected with a clear error message showing the doctor's actual hours.
    

                    
    Patient Deduplication
    
                    
    Patients are matched by email or phone. If the same patient books multiple times, they are recognized as one person — not counted as duplicates. The response includes is_returning: true/false so you know if this is a new or existing patient.
    

                    
    Your API Key
    
                    
    Your API key is permanent — it never expires and never changes, even if you log out. Each company has a unique key. Use it in the Authorization: Bearer header for all requests.
    

                
    
            
    

            
    
                
    API Endpoint
    
                POST /api/external/booking
                
    Headers:
    
                Authorization: Bearer YOUR_TOKEN
                Content-Type: application/json
            
    

            
    
                
    Create Appointment — Required JSON Body
    

    {
  "patient_name": "John Doe",           // Required — full name
  "patient_email": "john@example.com",   // Required — used for deduplication
  "patient_phone": "+1234567890",        // Required — used for deduplication
  "doctor_name": "Dr. Smith",            // Required — must match a doctor in your account
  "service": "Root Canal",               // Optional — if empty or omitted, will be set to "NA"
  "date": "2026-04-20",                  // Required — YYYY-MM-DD format
  "time": "14:00",                       // Required — HH:MM (24h format)
  "notes": "Patient prefers morning"     // Optional
}
    
                
    Duplicate patients (same email or phone) are automatically merged — they won't be counted as separate patients.

                Response Format & Error Codes
                
    
                    Success (201):
    
                    {"ok": true, "booking_id": 123, "patient_id": 45, "is_returning": false, "message": "Appointment synced successfully"}

    
                    Errors:
    
                    401 — Invalid or missing API key
    
                    400 — Missing required fields (patient_name, patient_email, patient_phone, doctor_name, date, time)
    
                    404 — Doctor not found in your account
    
                    409 — Time slot already booked for that doctor
